# Database Skill

## Instructions

1. **Schema design**
   - Define tables according to feature requirements
   - Ensure relationships (foreign keys) are correct
   - Include necessary fields, types, and constraints

2. **Table creation**
   - Use SQLModel (or appropriate ORM) for table definitions
   - Include default values and nullable settings
   - Prepare the database for multi-user support

3. **Migrations**
   - Generate migration scripts for schema changes
   - Ensure data integrity during updates
   - Version control all migrations for consistency

## Best Practices
- Keep table and column names clear and descriptive
- Enforce user isolation in multi-user environments
- Index frequently queried fields for performance
- Avoid redundant fields or tables
- Test schema changes before production

Use this skill whenever database schema, tables, or migrations need to be created or updated.
