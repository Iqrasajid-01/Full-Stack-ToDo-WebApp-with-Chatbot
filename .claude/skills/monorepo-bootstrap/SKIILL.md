# Monorepo Bootstrap

## Instructions

1. **Initialize monorepo**
   - Create a single root repository
   - Prepare the project for multi-agent workflows
   - Ensure compatibility with Spec-Kit Plus

2. **Project organization**
   - Establish clear boundaries between frontend, backend, and specifications
   - Standardize folder naming and hierarchy
   - Enable easy navigation for automated agents

3. **Spec-driven readiness**
   - Prepare repository for spec-first development
   - Ensure specs can be referenced across all layers
   - Support phase-based development structure

## Best Practices
- Prioritize clarity over complexity
- Keep structure minimal and predictable
- Design for scalability across phases
- Avoid feature or logic implementation at this stage

Use this skill only once at the beginning of the project lifecycle.
