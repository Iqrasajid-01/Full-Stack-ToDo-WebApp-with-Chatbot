# Frontend Pages & Components

## Instructions

1. **Page Layouts**
   - Structure pages with clear header, content, and footer sections
   - Use responsive grid and flex layouts
   - Ensure consistent spacing and alignment

2. **Components**
   - Build reusable components (buttons, forms, cards, lists)
   - Ensure proper props handling and state management
   - Components should be mobile-first and accessible

3. **Styling**
   - Apply consistent color scheme and typography
   - Use Tailwind CSS classes for rapid styling
   - Avoid inline styles, prefer utility classes and reusable styles

## Best Practices
- Keep component hierarchy simple and predictable
- Use semantic HTML wherever possible
- Ensure accessibility (a11y) and mobile responsiveness
- Maintain a centralized design pattern for consistency

Use this skill whenever creating new pages, UI components, or layouts in the frontend.
